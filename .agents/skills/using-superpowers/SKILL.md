# Superpowers Framework Skill

The goal of this skill is to explain the Superpowers framework and how to use it to achieve the best results.

## Overview

Superpowers is an agentic framework that provides a structured approach to software development for AI assistants. It's built around a set of discrete, composable skills that guide the assistant through the entire development lifecycle.

## Core Concepts

-   **Skills**: Reusable instructions for specific tasks (e.g., `brainstorming`, `writing-plans`, `executing-plans`).
-   **Agents**: Specialized AI entities that collaborate on complex tasks (e.g., `code-reviewer`).
-   **Methodology**: A test-driven, plan-driven approach to engineering.

## Standard Workflow

1.  **Brainstorm**: Explore ideas and identify the best approach.
2.  **Plan**: Create a detailed, actionable plan.
3.  **Execute**: Implement the plan, tracking progress and verifying every step.
4.  **Verify**: Perform final verification and testing.
5.  **Deliver**: Provide a summary of the accomplishments and any next steps.

## Guidelines

-   **Follow the methodology**: The framework's value comes from its structure. Don't skip steps.
-   **Document everything**: Use artifacts (`brainstorming_notes.md`, `implementation_plan.md`, `task.md`, `walkthrough.md`) to maintain transparency.
-   **Collaborate with the user**: The framework is designed for pair programming.
-   **Verify early and often**: Don't wait until the end to find bugs.
-   **Aesthetics and Quality**: Always aim for premium-quality code and design.
