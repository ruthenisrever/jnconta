# Brainstorming Skill

The goal of brainstorming is to generate a diverse range of ideas and potential solutions for a given problem or task. This step is crucial BEFORE creating a formal implementation plan, as it allows for exploration and discovery of better approaches.

## Process

1.  **Initial Assessment**: Briefly summarize the problem or task.
2.  **Idea Generation**: Generate as many ideas and approaches as possible, without judgement.
3.  **Feasibility & Impact Analysis**: For each significant idea, evaluate its technical feasibility and potential impact on the project.
4.  **Refinement**: Select the most promising ideas and refine them into a high-level approach.
5.  **Output**: A list of ideas, a selected approach, and a set of next steps (usually leading into `writing-plans`).

## Guidelines

-   **Be creative and open-minded**: Don't dismiss ideas too early.
-   **Think about edge cases**: Consider what could go wrong and how different approaches handle it.
-   **Focus on the USER's needs**: Ensure the selected approach directly addresses their core objective.
-   **Keep it high-level**: Avoid deep technical implementation details during brainstorming.
-   **Aesthetics matter**: If the task involves UI, brainstorm visual enhancements and modern design patterns.

## Output Structure

The result of this skill should be documented in a `brainstorming_notes.md` artifact or directly in the conversation.

## Next Steps

Once a high-level approach is agreed upon, proceed to the `writing-plans` skill.
